# FDS 학습 프레임워크
## 파산관재인 및 은행권 Fraud Detection System을 위한 프로그래밍·데이터 분석 학습 프레임워크

> **최종 목표**: 설명 가능한 FDS 포트폴리오를 통한 쿠팡·무신사 데이터 분석가/엔지니어 전환

---

## 📌 프로젝트 개요

물류 6년 경력 후 데이터 분석가로 전환하는 개발자가 구축하는 실무형 FDS 포트폴리오입니다.

**현실 문제를 코드로 번역하는 능력**을 중심으로, 파산관재인을 위한 재산 은닉 탐지 시스템과 Kaggle 은행권 이상거래 탐지 모델을 동시에 설계·구현합니다.

### 🎯 핵심 철학

- **문법 암기가 아니라 번역 능력 습득**: 프로그래밍 문법을 현실 문제 해결 도구로 학습
- **울트라러닝**: 스콧 영의 울트라러닝 방식으로 메타 학습 → 포커스 → 직접 학습 → 피드백 반복
- **설명 가능성**: 블랙박스 모델이 아닌, 관재인·사용자가 이해할 수 있는 시스템 설계
- **이산수학의 정당성**: 자료구조·알고리즘의 수학적 기초를 이해하며 최적화

---

## 📚 학습 자료 (2부 구성)

### **1부: 프로그래밍 문법 학습의 목적과 현실 문제 번역 사고틀**
[📄 01_프로그래밍_문법_사고틀.md](./01_프로그래밍_문법_사고틀.md)

**15개 섹션, 현실 문제 중심의 문법 재정의**

- 문법은 도구, 목적은 번역 능력
- 상태·자료구조·추상화·인터페이스의 실무 의미
- 파산관재인 FDS 사고틀 구체 예시
- 7단계 훈련 루틴 (문법 ↔ 현실 문제 연결)
- 일간·주간 학습 설계
- 좋은 코드의 기준 (특히 설명 가능성)

**대상**: 파이썬 입문자, 데이터 분석가 전환자

---

### **2부: 이산수학·자료구조·알고리즘의 상호 관계와 역사적 기원**
[📄 02_프로그래밍_문법_사고틀_확장편.md](./02_프로그래밍_문법_사고틀_확장편.md)

**왜 자료구조와 알고리즘이 필요한가를 수학적·역사적으로 설명**

- 이산수학: 정당성(Correctness)
- 자료구조: 충실성(Fidelity)
- 알고리즘: 도달 가능성(Feasibility)
- 튜링, 오일러, 폰 노이만, 커누스의 역사적 흐름
- 복잡도 분석과 실무 적용
- 파산관재인 FDS에 적용한 구체 예시

**대상**: 알고리즘 학습 중인 데이터 엔지니어, 포트폴리오 심화 단계

## 🎯 포트폴리오 프로젝트

### **프로젝트 1️⃣: 파산관재인 FDS (메인 포트폴리오)**
📂 [`bankruptcy-fds/`](./bankruptcy-fds/)

**재산 은닉, 편파 변제, 자금 레이어링 탐지**

**핵심 탐지 규칙:**
- **쪼개기 송금**: 파산 전 30일, 동일 수취인, 5회 이상, 500만원 이상
- **편파 변제**: 가족·지인에게 집중된 송금
- **레이어링**: 다중 계좌를 거친 자금 이동 경로 (그래프 알고리즘)

**학습 기간**: 12주 (1주: 모델 설계 → 4주: 규칙 구현 → 8주: 파이프라인/UI)

---

### **프로젝트 2️⃣: 은행권 FDS (Kaggle PaySim)**
📂 [`kaggle-bank-fds/`](./kaggle-bank-fds/)

**은행 이상거래 탐지 (규칙 기반 + 머신러닝)**

**파산 FDS와의 연계:**
- 같은 `shared.rules.BaseFraudRule` 상속
- 도메인만 다르고 사고틀은 동일
- 코드 재사용으로 울트라러닝 효과

**학습 기간**: 8주 (병렬 진행, 파산 FDS와 겹침)

---

## 🔗 공유 모듈 구조

### **`shared/` 패키지 (양 프로젝트가 함께 사용)**

**설계 철학**: 
- 두 도메인의 공통 로직은 shared에
- 도메인 특화 로직은 각 프로젝트의 src에 구현
- 코드 재사용으로 DRY(Don't Repeat Yourself) 원칙 적용

## 🛠 기술 스택

### **개발 환경**
- **언어**: Python 3.9+
- **IDE**: PyCharm (코드), DataSpell (Jupyter), DataGrip (SQL)
- **기기**: MacBook Air M5

### **데이터 처리**
- pandas (테이블형 데이터 변환)
- numpy (수치 계산)
- SQLAlchemy / pandas.read_sql (DB 쿼리)

### **머신러닝 & 모델링**
- scikit-learn (기본 분류 모델)
- XGBoost, LightGBM (부스팅)
- SHAP, LIME (설명 가능성)

### **시각화 & UI**
- Streamlit (설명형 대시보드)
- matplotlib, seaborn (통계 시각화)
- plotly (인터랙티브 그래프)

### **협업 & 배포**
- Git / GitHub (포트폴리오 관리)
- Docker (선택사항)
- Kaggle API (데이터셋, 제출)

### **1. 환경 설정**
```bash
cd ~/Documents/fds-learning-framework
source venv/bin/activate
pip install -r requirements.txt
```

### **2. Jupyter 노트북 시작 (EDA 단계)**
```bash
jupyter notebook bankruptcy-fds/notebooks/
```

### **3. 유닛 테스트 실행**
```bash
pytest bankruptcy-fds/tests/
```

### **4. Streamlit 대시보드 (UI 단계)**
```bash
streamlit run bankruptcy-fds/src/ui/streamlit_app.py
```
